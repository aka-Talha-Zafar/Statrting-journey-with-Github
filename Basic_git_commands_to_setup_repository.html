Common git/github comands to perform basic/useful functions on repositories


Cloning a repository

If you are a beginner it is advised to create a remote repository first and then clone it localy for editing.You can also create a repository localy first and then link it to remote one, but for a beginner this approach is not recomended. Cloning of github repository can be done by executing sigle line/simple command:
Command = git clone http/ssh_address_of_repo_you_want_to_clone


Git Configurations: (Configuring git with username and email)

To configure your name and gmail account with github repository run following commands on terminal after cloning the repository:

For user Name
git config --global user.name "Your github user name here"
For email
git config --global user.email "Your email address asociated to github"
Now to see all configurations use command
git config --list
To see only the user name and email only use
git config --global user.name
git config --global user.emai


Git status command

Git status command tells us about the present status of repository or code. Tells wether the code/repo is up to date or need any changes to be merged
git status
There are four types of status in Git:
1) Untracked (Any new file created that does not tracked by git yet.If you made any changes to this file it will still remains untracked not modified as this file is not tracked by git)
2) Modified (Shows that you have made any change in file which is not added/committed yet.)
3) Staged (Shows that file is ready to be committed. Means the changes you have made to file are added and now ready to be committed)
4) Unchanged (Means there is not any change made to file yet, everything is up to dated)


Add & Commit

Git add:

Git add is used to add new or changed files in your working directory to the git staging area.
Note: Git staging area acts as a middle ground between your working directory and your Git repository.This is where you tell Git, "I want to include these changes in the next commit."

Commands for adding files
git add "file name" (Used to add single or sepecific file at a time)
git add . (Used to add all updated files at once)

Commit:
Once you have add changes of files it is not updated yet, we use git commit command to update files or to save changes.It is like the snapshot or record of changes you have made yet.
Command = git commit -m "Description of changes/Meaningful Message"


Push Command

Push comand is used to upload commits/changes from local repo (your PC) to remote repo (Github)
Command = git push origin main
(Pushes your local "main" branch to the remote "origin/main".But it does not set up any upstream/tracking relationship.Next time you want to push, you'll need to specify the branch again.)
Command = git push -u origin main
(Pushes your local "main" branch to "origin/main", Also sets up your local branch to "track" the remote branch.After this, Git knows “Okay, when working on main, I should push/pull from origin/main by default.”)


Init Command

This command initializes a new local git repository in your current directory.Creates a hidden .git folder due to which your project folder becomes a local Git repository, but not yet connected to any remote like GitHub.
Command = git init

Command to link local repo to remote repo (Github)

This command is used to connect/associate your local repo with a remote one (usually on GitHub, GitLab, etc.).
It involves some basic steps which you should have to consider:
1) Create a local git repo
2) Create a remote repo on (Github) without adding a readme file
3) Copy the HTTPS/SSH address provided on remote repo
4) Execute the link command
Command = git remote add origin "HTTPS/SSH_link_of_remote_repo"

Command to verify remote repo

After linking the local repo with remote one you can check wether you linked to correct remote repo by using this command. It List down all configured remote repositories and their URLs.
Command = git remote -v

Command to check working branch

Shows all local branches in your repository and highlights the current working branch with an asterisk (*).
Command = git branch

Command to rename branch

Renames the current branch to main. The -M flag forces the rename, even if a branch named main already exists.This is done because Git automatically defines "master" as the main branch.But, main is the new convention (especially on GitHub).
Command = git branch -M main

Command to nevigate between branches

Switches to an existing branch named "branch-name" in your local repository.
Command = git checkout "branch-name"

Command to create new branch

Creates a new branch named "new-branch-name" and automatically switches to it.
Command = git checkout -b "new-branch-name"

Command to delet branch

Used to delete an existing repository.It will not delete current working branch, you have to nevigate to other branch which you dont want to delete.
Command = git branch -d "branch name" (Deletes the specified local branch, but only if it has been fully merged into your current branch.)
Command = git branch -D "branch name" (Used to force delete the branch even if the changes are not merged)

Command to compare two brnaches

This is a comparison command used to see differences between your current branch and another branch.SHows line by line changes between your current working branch (main) and specified "branch name".
Command = git diff "branch-name"

Command to merge two branches

This command is used to merge the specified branch (branch-name) into your current branch (main).
Command = git merge "branch-name"

Pull request

A Pull Request (PR) is a request to merge one branch into another in a remote Git hosting service (like GitHub, GitLab, Bitbucket).Basically, you tells the maintainer/owner of repo that, I have made some changes in my branch can you please review and merge them to main brach.

Pull command

This command fetches the latest changes from the main branch of the remote repo (Github/origin), and then merges them into your current local branch.
Command = git pull origin main

Merge conflicts

A merge conflict happens when Git can’t automatically decide how to combine changes made in different branches — because the same lines of a file have been edited in both branches.When ever a merge conflict occurs you got 4 different ways/options to merge the code:
1) Accept Current Change (It allows us to accept the changes of current/working branch)
2) Accept Incoming Change (It allows us to accept the changes coming from other branch (in which we are not present/working) )
3) Accept Both Changes (It allows us to accept changes of both branches)
4) Compare Changes (It opens a side-by-side view of both versions so you can visually inspect and edit the result manually.)

Commands to undo changes

If you’ve made changes to your repository and want to undo them, Git provides several commands and methods to help you revert or discard those changes.
1) Stagged/add changes
Command = git reset "file_name" (Used when you have made changes to single file or want to revert changes of single file)
Command = git reset (Used when you have to revert changes from multiple files)
2)Commited changes
Command = git reset HEAD~1 (Used to revert only 1 (one) commit back.)
Command = git reset "commit hash" (Used to revert back to specified commit "Hash value is the assigned code of specified commit".It deletes the recent commit in git but we can not see changes on VScode)
Command = git reset --hard "commit-hash" (Used to revert back to specified commit "Hash value is the assigned code of specified commit".It also show changes on VScode)

Command to see commit history

Displays the commit history of the current branch, showing all commits made — in reverse chronological order (most recent first).Display's all necessary info of commit including hash valus, author name, date of commit and etc.
Command = =git log

Fork

Fork is used to create a remote copy of other's repo under your GitHub account.Allows you for Open source collaboration, experimentation and independent development without affecting the original repo.You can fork other's repo under your account and after performing changes, create a new pull request which can be merged by maintainer of repository after reviewing it.

Smart Git Habits for Clean Collaboration


1) Always do a git pull before you start a new session of work, just to be synced.
2) Always create a new branch before making experimental changes. That way, your main branch remains clean and stable.
3) Always review your changes with git diff before committing.It prevents accidental commits of debug code or half-done work.
4) Pull before you push to avoid conflicts, as it keeps your changes compatible with the latest remote updates.
5) Use descriptive branch names, as clear and useful make collaboration cleaner.
